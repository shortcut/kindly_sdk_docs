KindlySDK

Kindly ChatBot SDK is a communications service provider. The SDK platform provides chat using the XMPP and HTTPS protocols.

Initialisation SDK

Initialize the SDK with the next credentials:

     | Name                    |           Type            |
     |_________________________|___________________________|
     | API key                 |                  String   |
     | App Cluster             |                  String   |
     | ChatBot key             |                  String   |
     | Code language (optional)|           ("en") String   |

    The ChatKindlySDK class is responsible for the initialisation of the SDK.

Requirements

The minimum requirements for Kindly ChatBot SDK are: Android 6.0 (API 23) Gradle plugin 7.2.1 Gradle 7.3.3

Authentication

Kindly supports authenticating users. When your users are authenticated, it's easier to identify them in the Kindly Inbox and you can create more personalized and powerful webhook integrations. To set this up you need to do a few things: - Initialize SDK ChatKindlySDK(Application, ApiKey, ChatBotKey", Cluster) - Connect to chat bot - Subscribe to message channel a callback to Kindly Chat initialization code. For more details read the the link (https://docs.kindly.ai/api/application)

Start chatting

Open the ChatBot via the function launchChat with a context parameters. The sdk will connected with API and establishing connection for transferring messages.

Image carousel

The SDK supports a feature showing a container (slideshow) of images or info that users can select by clicking a button that directs them forward or backward in the slideshow. A developer uploads images in the console the next formats: "gif", "jpeg", "jpg". Also, the developer can set a size of the images: SMALL - 218 dp, MEDIUM - 178 dp, BIG - 148 dp.

Attachments

A user can upload 5 files. The SDK supports: "gif", "jpeg", "jpg" and "pdf". The max size of the file is 5 MB. In one time a user can upload min 1 and max 5 files. For viewing a pdf file uses the app installed on the device.

Delete chat log

The current session wit the bot could be cleared by user. After removing logs the session reinitialised.
